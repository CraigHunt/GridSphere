Portlet Layout Components

Portlet layout components are the visual graphical components that get displayed to a client consisting of presentation information in an appropriate markup language (currently just HTML).

Portlet components are typically defined in an XML schema known as layout.xml and unmarshalled to Java objects using the Castor java data binding libraries.

All Layout components implement a {@link org.gridlab.gridsphere.layout.ComponentRender} interface responsible for rendering the presentation content. In addition, most concrete portlet components implement a {@link org.gridlab.gridsphere.layout.ComponentLifecycle} interface responsible for handling all component lifecyle methods such as initialization, shutdown, and performing actions.

The current list describes the concrete portlet components that are in use:

  • {@link org.gridlab.gridsphere.layout.PortletFrame}
  • {@link org.gridlab.gridsphere.layout.PortletGridLayout}
  • {@link org.gridlab.gridsphere.layout.PortletImage}
  • {@link org.gridlab.gridsphere.layout.PortletPanel}
  • {@link org.gridlab.gridsphere.layout.PortletTab}
  • {@link org.gridlab.gridsphere.layout.PortletTabbedPane}
  • {@link org.gridlab.gridsphere.layout.PortletText}
  • {@link org.gridlab.gridsphere.layout.PortletTitleBar}
